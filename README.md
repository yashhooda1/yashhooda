# Yash Hooda – Personal Portfolio
**Live:** [yashhooda1.vercel.app](https://yashhooda1.vercel.app)

A full-stack personal portfolio with a production-grade AI chatbot, live Strava training analytics, real-time flight tracking, network analysis tools, interactive snow & hike photo albums, live weather, and a secure AI gateway — all deployed on Vercel.

---

## Features

### 🤖 AI Chatbot (HoodaAgents)
- Powered by **Claude (Anthropic)** and **GPT-5.x (OpenAI)** with plug-and-switch model routing
- **RAG** (Retrieval Augmented Generation) via Upstash Vector — retrieves relevant context from a knowledge base
- **Persistent memory** via Upstash Redis — remembers past conversations per session (30 days)
- **Voice input** via Web Speech API + **voice output** via OpenAI TTS (nova voice)
- **Page control workflows** — bot can scroll to sections, open links, read live stats aloud
- **Chat history** — save, load, and delete past conversations
- **Secure AI Gateway** — 7-layer security architecture (see Security section)

### 🏃 Live Training Dashboard
- **Live Training Feed** — last 30 Strava activities with interactive route maps (Leaflet.js), pace, HR, suffer score, kudos
- **Strava Intelligence** — CTL/ATL/Form score, 8-week mileage bar chart, pace zone donut chart, predicted race times (Mile, 5K, 10K, Half, Marathon via Riegel formula), AI Coach insights generated by Claude
- Weekly mileage resets every Monday, counts only Run activities

### ✈️ Aviation Tracker
- Live US flights via **AviationStack API**
- Interactive Leaflet map with rotating plane icons
- Flight cards with altitude, speed, heading, climb rate
- Click any card to locate the plane on the map
- Auto-refreshes every 5 minutes

### 🔍 Network Analyzer
- **DNS Lookup** — A, AAAA, MX, TXT, NS, CNAME records
- **WHOIS** — domain registration, registrar, dates, nameservers via RDAP
- **Ping** — host reachability and response time simulation
- **Traceroute** — network path mapping
- **HTTP Headers** — response headers + security analysis (HSTS, CSP, X-Frame-Options, etc.)
- Terminal-style green-on-black output

### 🌤️ Weather Widget
- Live weather for visitor's current location via **Open-Meteo API** (no API key required)
- Reverse geocoding via Nominatim (OpenStreetMap)
- Shows temp, feels like, humidity, wind speed/direction, precipitation

### ❄️ Snow Album
- Categorized Texas snow photos and NYC Blizzard photos (Historic February 2026 Nor'easter)
- Fullscreen lightbox with keyboard navigation (← → Esc)
- Snow highlight reel video

### 🏔️ Hike Journal
- Morning Hike around Lake Monarch — Granby, CO (Arapaho National Forest)
- Route map, 3D view, activity summary, photo journal

## Deployment

### 1. Clone and push to GitHub
```bash
git clone https://github.com/yashhooda1/yashhooda1.github.io
cd yashhooda1.github.io
git add .
git commit -m "your message"
git push
```
Vercel auto-redeploys on every push to `main`.

### 2. Add environment variables
Go to [vercel.com/dashboard](https://vercel.com/dashboard) → your project → **Settings** → **Environment Variables** → add each key above.

### 3. Local development
```bash
npm install -g vercel
vercel dev
```
Create a `.env.local` file (never commit this):
```

```

## API Architecture

```
Browser
  │
  ├── POST /api/chat        → Multi-model AI router (Anthropic / OpenAI)
  │                              ↓ RAG (Upstash Vector)
  │                              ↓ Memory (Upstash Redis)
  │                              ↓ Security layers
  │
  ├── GET  /api/strava      → Strava OAuth + activity shaping
  ├── GET  /api/analytics   → Training analytics + Claude AI insights
  ├── POST /api/tts         → OpenAI TTS audio stream
  ├── GET  /api/flights     → AviationStack live flights
  ├── POST /api/network     → DNS/WHOIS/ping/traceroute/headers
  └── GET/POST /api/history → Chat history (Upstash Redis)
```

## Security Architecture

The chatbot implements a **7-layer secure AI gateway**:

| Layer | Description |
|-------|-------------|
| 1 — Input Validation | Rejects malformed messages, oversized inputs, invalid roles |
| 2 — Jailbreak Prevention | 25+ regex patterns detect prompt injection, DAN attacks, persona hijacking, system prompt extraction |
| 3 — RAG Sanitization | Strips injected instructions from vector DB chunks before they reach the model |
| 4 — Output Filtering | Scans every reply for API keys, tokens, env vars before sending to user |
| 5 — Tool Permission Boundaries | RAG, memory, image tools each have explicit permission checks and configurable limits |
| 6 — Rate Limiting | 20 requests/minute per session — prevents abuse and API cost attacks |
| 7 — Instruction Hierarchy | SYSTEM PROMPT > RAG CONTEXT > MEMORY > USER INPUT — user messages can never override system rules |
